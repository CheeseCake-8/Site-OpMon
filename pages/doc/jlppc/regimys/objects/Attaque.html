Classe jlppc.regimys.objects.Attaque

La classe qui définit une espèce de Pokémon.

public abstract class Attaque extends RegimysObject implements Serializable

  • Variables
  • Classes internes
  • Constructeurs
  • Methodes

Variables :
  • protected String nom : Le nom de l'attaque
  • protected int puissance : La puissance de l'Attaque
  • protected int priorite : Le niveau de priorité de l'Attaque
  • protected short precision : La precision de l'Attaque
  • protected Type type : Le type de l'Attaque
  • protected boolean special : true si l'attaque est spéciale, false si elle est physique.
  • protected boolean status : Si c'est une atatque de status ou non
  • protected int part : La partie de l'attaque. Variable utilisée lors des attaques a plusieurs parties.
  • protected int pp : Les pp actuels de l'Attaque
  • protected int ppMax : Les pp max de l'attaque
  • protected int chanceDeCoups : La chance de coups critique (1 chance sur cette variable)
  • protected boolean rateJamais : Si l'attaque ne rate jamais
  • protected int pvPerdus : Contient les PV perdus par l'attaque, après le calcul des dégats
Classes internes :
  • Aucune classe interne n'est présente dans cette classe.
Constructeurs :
  • protected Attaque(String nom, int puissance, Type type, int precision, boolean special, boolean status, int chanceDeCoups, boolean rateJamais, int ppMax) : Constructeur normal.
  • protected Attaque(String nom, int puissance, Type type, int precision, boolean special, boolean status, int chanceDeCoups, boolean rateJamais, int ppMax, int priorite) : Constructeur avec la priorite.
Methodes :
  • Je ne documente pas les setters et getters
  • public final void attack(Pokemon atk, Pokemon def) : Lance une attaque du pokemon attaquant (atk) contre le pokemon defenseur (def)
  • AVERTISSEMENT : Dans les prochaines methodes, il y aura les paramètres de type Pokemon atk et def qui ne seront pas documentés car il signifient la meme chose que l'attaque d'au dessus
  • public abstract void effetAvant(Pokemon atk, Pokemon def) : Methode a redefinir permettant d'effectuer une action avant le calcul des dégats.
  • public abstract void effetApres(Pokemon atk, Pokemon def) : Methode a redefinir permettant d'effectuer une action après le calcul des dégats.
  • public void siEchoue(Pokemon atk, Pokemon def) : methode vide dont la redefinition n'est pas obligatoire permettant d'effectuer une action lorsque l'attaque echoue.
